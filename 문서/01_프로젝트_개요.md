# 프로젝트 개요

## 1. 프로젝트 목적

본 프로젝트는 반복적으로 확인해야 하는 기술 뉴스를 자동으로 수집하고 요약하여 Notion 데이터베이스에 저장하는 것을 목표로 한다. 이를 통해 사용자는 여러 뉴스 페이지를 직접 확인하지 않고도, 관심 분야인 로봇 관련 기사만 선별하여 빠르게 확인할 수 있다.

자동화 대상은 AI타임스의 전체 기사 RSS 피드이다. Make.com의 메인 뉴스 요약 시나리오가 `Asia/Seoul` 기준 매일 09:00에 피드를 확인하고, 제목·설명·요약에 `로봇`이 포함된 기사를 선별한다. 이후 원문 링크를 기준으로 기존 저장 여부를 확인한 뒤, 신규 기사에 한해서만 원문 본문 수집과 Gemini 요약을 수행한다. 오류 모니터링 시나리오는 매일 09:10에 실행되어 최종 실패 건을 확인한다.

## 2. 해결하고자 한 문제

기술 동향 조사는 여러 뉴스 사이트를 반복적으로 방문하고, 관련 기사를 찾은 뒤, 핵심 내용을 정리하여 협업 도구에 옮기는 작업을 요구한다. 수작업 방식에는 다음과 같은 문제가 있다.

- 관련 없는 기사까지 직접 확인해야 하므로 탐색 시간이 길다.
- 동일 기사를 여러 번 확인하거나 저장할 수 있다.
- 기사 본문을 복사하고 요약하는 과정이 반복적이다.
- 제목, 링크, 발행일시 등의 속성을 일관되게 관리하기 어렵다.
- 수집과 정리에 시간을 사용하면 실제 분석에 투입할 시간이 줄어든다.

## 3. 자동화 필요성

RSS 수집, 주제 필터링, 중복 확인, 본문 추출, AI 요약, 데이터베이스 저장을 하나의 흐름으로 연결하면 반복 작업을 줄이고 결과 형식을 일정하게 유지할 수 있다. 특히 중복 확인을 HTTP 요청과 Gemini 호출보다 앞에 배치하여 동일 기사에 대한 불필요한 처리 시간과 API 비용을 줄였다.

또한 Make의 스케줄, Retry, Incomplete executions를 활용하면 일시적인 오류를 자동으로 복구하고 최종 실패 건도 추적할 수 있다. 따라서 사람이 매 실행을 확인하지 않아도 수집부터 저장까지 안정적으로 이어지는 구조를 구현할 수 있다.

## 4. 최종 결과물

최종 시스템은 다음 결과물을 생성한다.

1. `Asia/Seoul` 기준 매일 09:00에 자동 실행되는 Make 메인 뉴스 요약 시나리오
2. AI타임스 RSS에서 `로봇` 관련 기사만 선별하는 필터
3. 원문 링크를 기준으로 한 Notion 중복 확인 구조
4. 기사 원문 HTML에서 `<p>` 문단을 추출·병합·정제하는 본문 처리 구조
5. Gemini가 생성한 3줄 이내의 한국어 요약문
6. 제목, 요약문, 원문 링크, 발행일시, 상태가 매핑된 Notion 데이터베이스 항목
7. 매일 09:10에 실행되며 Retry와 Incomplete executions를 기반으로 동작하는 오류 추적 구조
8. 워크플로우 구조, 필터링 기준, 오류 처리 정책, 팀 역할을 설명하는 제출 문서

워크플로우와 저장 결과 스크린샷은 `스크린샷/` 및 `결과_예시/`에 정리하였으며, Make 시나리오 공유 링크와 Blueprint JSON은 `Make_시나리오/`에서 확인할 수 있다.

## 5. 보안 원칙

본 저장소에는 API 키, 인증 토큰, Authorization 헤더 등 민감한 인증 정보는 포함하지 않는다. Make, Notion, Gemini 등의 실제 연결 정보는 제출용 스크린샷에서도 마스킹하여 관리한다. 개인 이메일과 비공개 링크도 문서에 기록하지 않는다.
