# AI 뉴스 자동 수집·요약·저장 시스템

**Make.com 기반 RSS → AI 요약 → Notion 저장 자동화 워크플로우**

---

## 1. 프로젝트 개요

AI 전문 매체 **AI타임스(aitimes.com)**의 RSS 피드를 매일 자동으로 수집해, "로봇" 키워드가 포함된 기사만 골라낸 뒤, 중복을 걸러 Claude AI로 3줄 요약하고 Notion 데이터베이스에 자동 저장하는 시스템이다.

사람의 개입 없이 **매일 정해진 시각에 자동 실행**되며, 일시적 오류가 발생해도 자동으로 재시도하는 예외처리까지 갖췄다.

### 핵심 목표

- 관심 주제(로봇) 기사를 매일 자동으로 모은다
- AI가 핵심만 3줄로 요약해 읽는 시간을 줄인다
- 같은 기사가 중복 저장되지 않게 한다
- 실패해도 시스템이 멈추지 않고 자동 복구한다

---

## 2. 전체 구조

```
[매일 08:00 자동 실행]
        │
        ▼
① RSS 수집 (모듈 10)
        │
        ▼
② 로봇 필터 ─── "로봇" 없으면 여기서 중단
        │
        ▼
③ Notion 중복 확인 (모듈 7)
        │
        ▼
④ 중복 필터 ─── 이미 있으면 여기서 중단
        │
        ▼
⑤ Claude AI 요약 (모듈 14) ──실패──▶ [재시도 2회 / 15분 간격]
        │
        ▼
⑥ Notion 저장 (모듈 8)
```

데이터가 위에서 아래로 흐르며, **두 개의 필터**가 관문 역할을 한다. 키워드에 안 맞거나 이미 저장된 기사는 중간에 걸러져 다음 단계로 넘어가지 않는다.

---

## 3. 단계별 상세 설명

### ① RSS 수집 — `RSS (모듈 10)`

AI타임스의 전체 기사 RSS 피드에서 최신 기사 목록을 가져오는 출발점이다.

| 설정 항목 | 값 |
|---|---|
| URL | `https://www.aitimes.com/rss/allArticle.xml` |
| Date from (수집 시작일) | 지정 날짜 이후 기사만 |
| Maximum number of returned items | `10` |

한 번 실행에 **최신 기사 최대 10개**를 가져온다. Date from으로 기준 날짜를 정해, 그 이후에 올라온 기사만 처리한다.

---

### ② 로봇 필터 — `로봇 필터`

RSS로 가져온 기사 중 **제목에 "로봇"이 들어간 기사만** 통과시키는 관문이다.

| 설정 항목 | 값 |
|---|---|
| 조건 대상 | 기사 제목(Title) |
| 연산자 | Contains (case insensitive) — 대소문자 구분 없이 포함 |
| 비교값 | `로봇` |

"로봇"이 제목에 없는 기사는 여기서 걸러져 더 이상 진행되지 않는다. 덕분에 관심 주제만 선별된다.

---

### ③ Notion 중복 확인 — `Notion (모듈 7) · Search Objects`

새로 들어온 기사가 **이미 Notion에 저장돼 있는지** 검색하는 단계다. 같은 기사를 두 번 저장하지 않기 위한 사전 점검이다.

| 설정 항목 | 값 |
|---|---|
| Connection | My Notion Internal connection |
| Search Objects | Data Sources |
| Query Search | `10.URL` (RSS 기사의 URL로 검색) |
| Sort | Last edited time – Descending |

기사의 **URL을 검색어로 사용**해 Notion DB 안에 같은 URL이 있는지 찾는다. URL은 기사마다 고유하므로 중복 판별 기준으로 적합하다.

---

### ④ 중복 필터 — `중복 필터`

앞 단계의 검색 결과를 보고 **처음 보는 기사만** 통과시킨다.

| 설정 항목 | 값 |
|---|---|
| 조건 대상 | 검색된 결과 개수 (Total number of bundles) |
| 연산자 | Numeric: Equal to |
| 비교값 | `0` |

검색 결과가 **0개 = Notion에 없는 새 기사**이므로 통과시킨다. 1개 이상이면 이미 저장된 기사이므로 여기서 중단한다. 이 필터가 중복 저장을 막는 핵심 장치다.

---

### ⑤ AI 요약 — `Anthropic Claude (모듈 14) · Create a Prompt`

새 기사 본문을 Claude AI에게 보내 **한국어 3줄 요약**을 받는다.

| 설정 항목 | 값 |
|---|---|
| Connection | My Anthropic Claude connection |
| Model | Claude Haiku 4.5 |
| Max Tokens | `300` |
| Role | User |
| Input Type | Single string |
| Content (프롬프트) | `다음 기사를 한국어 3줄 이내로 핵심만 요약해줘. 군더더기 없이 내용만:` + `10.Description` |

프롬프트 안에 RSS 기사의 본문(`Description`)을 넣어 전달한다. 빠르고 경제적인 **Claude Haiku 4.5** 모델을 사용했고, 요약은 짧으므로 Max Tokens는 300으로 제한했다.

---

### ⑥ Notion 저장 — `Notion (모듈 8) · Create a Database Item (Legacy)`

요약이 끝난 기사를 Notion 데이터베이스에 **새 항목으로 저장**하는 마지막 단계다.

| 설정 항목 | 값 |
|---|---|
| Connection | My Notion Internal connection |
| Database ID | `38a530bc8ac0807b8dbbe2b1deb0466b` |

**필드 매핑 (Notion 속성 ← 입력 데이터):**

| Notion 속성 | 매핑된 값 | 의미 |
|---|---|---|
| Name | `10.Title` | 기사 제목 |
| 요약 | `14.Text Response` | Claude가 생성한 요약 |
| URL | `10.URL` | 기사 링크 |
| 날짜 (Start Time) | `10.Date created` | 기사 작성일 |

RSS에서 가져온 제목·URL·날짜와 Claude가 만든 요약을, Notion DB의 각 속성에 정확히 연결했다.

---

## 4. 예외처리 — 자동 재시도

### `Retry (모듈 15)` — Claude 요약 실패 대비

AI 요약 단계(모듈 14)는 API 호출이라 일시적으로 실패할 수 있다(한도 초과, 네트워크 지연 등). 이를 대비해 **에러 핸들러로 자동 재시도(Retry)**를 붙였다.

| 설정 항목 | 값 |
|---|---|
| Retry automatically | Yes |
| Number of retries | `2` |
| Minutes between retries | `15` |

요약이 실패하면 **15분 간격으로 최대 2번까지 다시 시도**한다. 일시적 오류는 잠시 후 풀리는 경우가 많으므로, 재시도만으로 대부분 자동 복구된다.

> **설계 노트:** 재시도는 일시적 실패가 잦은 **AI 요약(Claude) 단계에만** 적용했다. Notion 저장 단계에도 한때 재시도를 붙였으나, 저장이 성공했는데도 응답 지연을 실패로 오인해 같은 기사가 여러 번 저장되는 문제가 있어 제거했다. 안정적인 단계에는 재시도를 두지 않는 것이 오히려 깔끔하다.

---

## 5. 자동 실행 스케줄

| 설정 항목 | 값 |
|---|---|
| 실행 주기 | 매일 (Daily) |
| 실행 시각 | 오전 08:00 |
| 시간대 | Asia/Seoul |

스케줄을 켜두면 **매일 아침 8시에 사람 없이 자동으로** 전체 흐름이 한 번 돌아간다. 밤사이 올라온 로봇 관련 기사가 아침에 요약된 채로 Notion에 정리돼 있다.

---

## 6. 사용한 도구·기술 요약

| 구분 | 사용 도구 |
|---|---|
| 자동화 플랫폼 | Make.com |
| 데이터 소스 | AI타임스 RSS 피드 |
| AI 요약 | Anthropic Claude (Haiku 4.5) |
| 저장소 | Notion 데이터베이스 |
| 연동 방식 | Notion Internal Integration, Anthropic API |

### 구현한 핵심 개념

- **필터링**: 키워드(로봇) 기반 선별 + 중복(URL) 제거
- **AI 연동**: 외부 API(Claude)에 데이터를 보내 가공된 결과 수신
- **데이터 매핑**: 한 모듈의 출력을 다른 모듈의 입력으로 연결
- **예외처리**: 에러 핸들러를 통한 자동 재시도
- **스케줄링**: 정해진 시각의 무인 자동 실행

## 7. 동작 한눈에 보기

> 매일 아침 8시, 시스템이 AI타임스에서 최신 기사 10개를 가져온다. 그중 제목에 "로봇"이 든 기사만 골라, 이미 저장한 적 있는지 URL로 확인한다. 처음 보는 기사라면 Claude AI가 한국어 3줄로 요약하고, 제목·요약·링크·날짜를 Notion에 깔끔하게 저장한다. AI 요약이 일시적으로 실패하면 15분 간격으로 두 번까지 알아서 다시 시도한다.
