# 주제 필터링 기준

## 1. 최종 주제

- RSS 출처: `https://www.aitimes.com/rss/allArticle.xml`
- 최종 키워드: `로봇`
- 검사 대상: RSS의 제목, 설명, 요약

## 2. `로봇` 키워드 선택 이유

AI타임스는 AI 전반을 다루는 기술 전문 뉴스 매체이기 때문에, 단순히 'AI'를 기준으로 필터링하면 반도체, 정책, 투자, 서비스, 생성형 AI 등 다양한 분야의 기사가 함께 수집되는 문제가 있었다. 따라서 보다 정확한 자동화와 일관된 데이터 수집을 위해서는 하나의 구체적인 기술 분야를 선정하는 것이 적합하다고 판단하였다. 그 결과 '로봇'을 최종 주제로 선택하였다.

로봇은 AI 기술이 실제 산업 현장과 물리적 장치에 적용되는 대표 분야이다. 휴머노이드, 산업용 로봇, 물류 자동화, 자율주행 로봇 등 비교적 명확한 범위의 기사를 확보할 수 있어 프로젝트의 데이터 주제를 일관되게 유지하기에 적합하다.

## 3. 필터 조건

Make의 RSS 모듈 다음 연결 구간에 OR 조건 필터를 배치한다.

```text
Title contains "로봇"
OR
Description contains "로봇"
OR
Summary contains "로봇"
```

세 필드 중 하나에만 키워드가 있어도 통과시키는 이유는 RSS 제공 형식이나 기사에 따라 핵심 주제가 제목이 아닌 설명·요약에 나타날 수 있기 때문이다. 반대로 세 필드 어디에도 `로봇`이 없으면 프로젝트 주제와 직접 관련되지 않은 기사로 판단하여 이후 단계를 실행하지 않는다.

## 4. 필터 배치 이유

주제 필터를 Notion 검색, HTTP 요청, Gemini 호출보다 먼저 배치하면 관련 없는 기사에 대한 후속 처리를 차단할 수 있다. 이 구조는 처리 시간과 API 호출을 줄이고, Notion 데이터베이스에 로봇 분야의 기사만 일관되게 축적되도록 한다.

실제 필터 조건과 설정 화면은 `스크린샷/03_로봇_기사_필터/`에서 확인할 수 있다.
